# GA4イベントトラッキング仕様書

## 概要
このドキュメントは、沖縄観光マップアプリケーションに実装されたGA4イベントトラッキングの完全な仕様を記載しています。全14種類のイベントがユーザー行動の詳細な分析を可能にします。

---

## イベント一覧

### イベント名: scroll_depth
**送信タイミング:** ユーザーがページをスクロールし、特定の深度に到達したとき
**送信条件:** 25%、50%、75%、90%の各深度に初めて到達したときのみ送信（重複送信なし）

| パラメータ名 | データ型 | 値の例 | 説明 |
|------------|---------|--------|------|
| depth_percentage | number | 25 | スクロール深度（%）。25, 50, 75, 90のいずれか |
| language | string | "ja" | 現在選択されている言語（ja / zh-TW / en） |

---

### イベント名: external_link_click
**送信タイミング:** ユーザーが外部リンクをクリックしたとき
**送信条件:** Google Maps方向案内リンク、ガイドブックリンクなどの外部サイトへのリンククリック時

| パラメータ名 | データ型 | 値の例 | 説明 |
|------------|---------|--------|------|
| link_url | string | "https://google.com/maps/dir/..." | クリックされたリンクのURL |
| link_text | string | "Get Directions" | リンクのテキスト内容 |
| language | string | "ja" | 現在選択されている言語 |

---

### イベント名: engagement_time
**送信タイミング:** ユーザーがアクティブな状態から非アクティブになったとき
**送信条件:** 最後の操作から3分間（180秒）操作がない場合、またはページ離脱時

| パラメータ名 | データ型 | 値の例 | 説明 |
|------------|---------|--------|------|
| engaged_seconds | number | 245 | アクティブエンゲージメント時間（秒）。非アクティブ時間は除外 |
| language | string | "ja" | 現在選択されている言語 |

---

### イベント名: search_result
**送信タイミング:** ユーザーが検索を実行したとき
**送信条件:** 検索ボックスに1文字以上入力された時点で即座に送信

| パラメータ名 | データ型 | 値の例 | 説明 |
|------------|---------|--------|------|
| search_query | string | "首里城" | ユーザーが入力した検索キーワード |
| result_count | number | 5 | 検索結果の件数 |
| language | string | "ja" | 現在選択されている言語 |

---

### イベント名: page_engagement
**送信タイミング:** ページ離脱時（beforeunloadイベント）
**送信条件:** 初回スクロールと初回インタラクションの両方が記録されている場合のみ送信

| パラメータ名 | データ型 | 値の例 | 説明 |
|------------|---------|--------|------|
| timeToFirstScroll | number | 3 | ページ読み込みから初回スクロールまでの時間（秒） |
| timeToFirstInteraction | number | 5 | ページ読み込みから初回インタラクションまでの時間（秒） |
| scrollSpeed | string | "normal" | スクロール速度（"slow" / "normal" / "fast" / "none"） |
| interactionCount | number | 15 | ページ内での総操作回数（クリック、スクロールなど） |
| scrollStopCount | number | 8 | スクロールが停止した回数 |
| language | string | "ja" | 現在選択されている言語 |

---

### イベント名: content_readability
**送信タイミング:** ページ離脱時（beforeunloadイベント）
**送信条件:** 常に送信

| パラメータ名 | データ型 | 値の例 | 説明 |
|------------|---------|--------|------|
| estimatedReadRate | number | 85 | 推定読了率（%）。スクロール深度と滞在時間から算出 |
| scrollDepth | number | 75 | 離脱時点でのスクロール深度（%） |
| timeSpent | number | 120 | ページ滞在時間（秒） |
| readingSpeed | string | "normal" | 読書速度（"slow" / "normal" / "fast"）。スクロール深度/時間比で判定 |
| language | string | "ja" | 現在選択されている言語 |

---

### イベント名: viewport_tracking
**送信タイミング:** セクションがビューポートから外れたとき、またはページ離脱時
**送信条件:** Intersection Observerで監視中のセクションが1秒以上表示された場合、またはスキップされた場合

| パラメータ名 | データ型 | 値の例 | 説明 |
|------------|---------|--------|------|
| sectionName | string | "main-map" | セクション名（data-section-name属性から取得） |
| visibilityTime | number | 45 | セクションが表示されていた時間（秒）。スキップされた場合は0 |
| wasSkipped | boolean | false | セクションがスキップされた（表示されなかった）かどうか |
| language | string | "ja" | 現在選択されている言語 |

---

### イベント名: session_behavior
**送信タイミング:** ページ離脱時（beforeunloadイベント）
**送信条件:** 常に送信

| パラメータ名 | データ型 | 値の例 | 説明 |
|------------|---------|--------|------|
| pagesViewed | number | 1 | セッション内で閲覧したページ数（現状は常に1） |
| spotsViewed | array | ["首里城", "美ら海水族館", "国際通り"] | 閲覧したスポット名の配列（最大5件） |
| userIntent | string | "planning" | 推定されるユーザー意図（"researching" / "planning" / "comparing"） |
| language | string | "ja" | 現在選択されている言語 |

**userIntent判定ロジック:**
- `comparing`: 5箇所以上閲覧 & 120秒以上滞在
- `planning`: 3箇所以上閲覧 & 60秒以上滞在
- `researching`: 上記以外

---

### イベント名: device_context
**送信タイミング:** ページ読み込み時（アプリケーション起動直後）
**送信条件:** 常に送信

| パラメータ名 | データ型 | 値の例 | 説明 |
|------------|---------|--------|------|
| time_of_day | string | "afternoon" | 時間帯（"morning" / "afternoon" / "evening" / "night"） |
| day_type | string | "weekday" | 曜日タイプ（"weekday" / "weekend"） |
| is_returning | boolean | true | リピーター訪問かどうか（localStorage判定） |
| visit_count | number | 3 | 訪問回数（localStorage管理） |
| language | string | "ja" | 現在選択されている言語 |

**time_of_day判定基準:**
- `morning`: 5:00-11:59
- `afternoon`: 12:00-16:59
- `evening`: 17:00-20:59
- `night`: 21:00-4:59

---

### イベント名: long_stay_user
**送信タイミング:** ページ滞在時間が60秒に達したとき
**送信条件:** 60秒滞在時に1回のみ送信（セッションごと）

| パラメータ名 | データ型 | 値の例 | 説明 |
|------------|---------|--------|------|
| stayDuration | number | 60 | 滞在時間（秒）。送信時は必ず60秒以上 |
| copyCount | number | 2 | コピー操作の実行回数 |
| tabSwitchCount | number | 1 | タブ切り替え回数（visibilitychangeイベント） |
| language | string | "ja" | 現在選択されている言語 |

---

### イベント名: exit_intent
**送信タイミング:** マウスカーソルがページ上部（Y座標 < 0）を離れたとき
**送信条件:** PCブラウザのみ（mouseleaveイベント）

| パラメータ名 | データ型 | 値の例 | 説明 |
|------------|---------|--------|------|
| scrollDepth | number | 45 | 離脱意図検知時のスクロール深度（%） |
| timeSinceLastAction | number | 15 | 最後の操作からの経過時間（秒） |
| trigger | string | "mouse_leave_top" | 離脱意図のトリガー種別 |
| language | string | "ja" | 現在選択されている言語 |

---

### イベント名: interaction_chain
**送信タイミング:** ページ離脱時（beforeunloadイベント）
**送信条件:** 1つ以上のインタラクションが記録されている場合のみ送信

| パラメータ名 | データ型 | 値の例 | 説明 |
|------------|---------|--------|------|
| actionSequence | array | ["scroll", "click", "external_link_click", "copy"] | ユーザー操作の順序配列（最大20アクション） |
| totalDuration | number | 180 | 最初の操作から最後の操作までの経過時間（秒） |
| language | string | "ja" | 現在選択されている言語 |

---

### イベント名: pin_click
**送信タイミング:** ユーザーが地図上のピンをクリックしたとき
**送信条件:** 常に送信

| パラメータ名 | データ型 | 値の例 | 説明 |
|------------|---------|--------|------|
| pinName | string | "首里城" | クリックされたピンの名称 |

---

### イベント名: affiliate_link_click
**送信タイミング:** ユーザーがアフィリエイトリンクをクリックしたとき
**送信条件:** 常に送信

| パラメータ名 | データ型 | 値の例 | 説明 |
|------------|---------|--------|------|
| link_url | string | "https://example.com/..." | アフィリエイトリンクのURL |
| location_name | string | "沖縄美ら海水族館" | スポットの場所名 |
| pin_title | string | "美ら海水族館" | ピンのタイトル |
| link_type | string | "official_site" | リンクの種別（official_site / booking / etc） |

## GA4での確認方法

### 1. リアルタイムレポートでの確認手順

1. **GA4管理画面にアクセス**
   - Google Analytics 4プロパティを開く
   - 左メニューから「レポート」→「リアルタイム」を選択

2. **イベント名で確認**
   - 「イベント名」カードでカスタムイベント（scroll_depth、page_engagement等）が表示されることを確認
   - 各イベント名をクリックすると、そのイベントのパラメータ詳細が表示される

3. **デバッグモードでの確認（推奨）**
   - ブラウザコンソール（F12）を開く
   - 各アクションを実行して「🎯 GTM Event Sent:」ログを確認
   - dataLayerに正しくイベントが追加されているか確認

### 2. カスタムディメンション設定が必要なパラメータ

以下のパラメータは、GA4でレポートとして活用するためにカスタムディメンションとして登録することを推奨します：

#### 必須設定（高優先度）
| パラメータ名 | スコープ | 説明 |
|------------|---------|------|
| language | イベント | ユーザーの言語設定。セグメント分析に必須 |
| user_intent | イベント | ユーザー意図。行動パターン分析に重要 |
| time_of_day | イベント | 時間帯別の行動分析に使用 |
| is_returning | イベント | 新規/リピーター分析に使用 |

#### 推奨設定（中優先度）
| パラメータ名 | スコープ | 説明 |
|------------|---------|------|
| scrollSpeed | イベント | スクロール速度の分析 |
| readingSpeed | イベント | コンテンツ読了速度の分析 |
| day_type | イベント | 平日/週末での行動差異分析 |
| trigger | イベント | 離脱意図のトリガー分析 |

#### 設定手順
1. GA4管理画面 → 「管理」→「カスタム定義」
2. 「カスタムディメンションを作成」をクリック
3. ディメンション名、スコープ、イベントパラメータを入力
4. 保存

### 3. 推奨されるカスタムレポート設定

#### レポート1: ユーザーエンゲージメント分析
**目的:** ユーザーの滞在時間と行動パターンを分析

- **ディメンション:**
  - language（言語）
  - user_intent（ユーザー意図）
  - is_returning（リピーター判定）

- **指標:**
  - engagement_time（平均エンゲージメント時間）
  - interaction_count（平均インタラクション回数）
  - scroll_depth（平均スクロール深度）

#### レポート2: コンテンツパフォーマンス
**目的:** コンテンツの読了率とスポット人気度を分析

- **ディメンション:**
  - spots_viewed（閲覧されたスポット）
  - language（言語）

- **指標:**
  - estimated_read_rate（平均読了率）
  - time_spent（平均滞在時間）
  - pin_click（ピンクリック数）

#### レポート3: 検索行動分析
**目的:** ユーザーの検索行動と検索効果を測定

- **ディメンション:**
  - search_query（検索キーワード）
  - language（言語）

- **指標:**
  - result_count（検索結果数）
  - search_result（検索実行回数）

#### レポート4: 時間帯別・訪問者別分析
**目的:** 訪問タイミングとリピーター行動の違いを把握

- **ディメンション:**
  - time_of_day（時間帯）
  - day_type（平日/週末）
  - is_returning（リピーター判定）

- **指標:**
  - セッション数
  - engagement_time（エンゲージメント時間）
  - user_intent（ユーザー意図の分布）

#### レポート5: 離脱・コンバージョン分析
**目的:** 離脱ポイントとコンバージョン（アフィリエイトクリック）を分析

- **ディメンション:**
  - scroll_depth（離脱時のスクロール深度）
  - trigger（離脱トリガー）

- **指標:**
  - exit_intent（離脱意図検知数）
  - affiliate_link_click（アフィリエイトクリック数）
  - external_link_click（外部リンククリック数）

### 4. BigQueryエクスポート推奨設定

長期的なデータ分析とより詳細な分析を行う場合、BigQueryへのエクスポートを推奨します：

- **actionSequence配列の分析:** ユーザーの操作順序パターンをSQL分析
- **spots_viewed配列の分析:** 人気スポットの組み合わせ分析
- **時系列分析:** 訪問回数と行動変化の相関分析

### 5. アラート設定推奨

以下の指標で異常値を検知するアラートを設定：

1. **engagement_time急減:** 平均エンゲージメント時間が前週比30%以上減少
2. **exit_intent急増:** 離脱意図検知が前週比50%以上増加
3. **search_result 0件急増:** 検索結果0件が前週比30%以上増加（コンテンツ不足の兆候）

## 注意事項

1. **個人情報保護:** 本実装では個人を特定できる情報は一切収集していません（GDPR対応）
2. **パフォーマンス:** throttle/debounceを適用し、パフォーマンスへの影響を最小化
3. **localStorage使用:** リピーター判定と訪問回数管理にlocalStorageを使用（クッキー不使用）
4. **エラーハンドリング:** すべてのイベント送信にtry-catchを実装
